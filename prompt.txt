Ты работаешь с репозиторием Flask-проекта `excel_converter`.

Цель: довести текущий проект до состояния коммерческого MVP, который выглядит как готовый B2B-продукт “ИИ-помощник для обработки сложных Excel-файлов”, а не как учебный прототип.

Проект уже умеет:

* загружать `.xlsx` / `.xls`;
* использовать Flask;
* работать с сохранёнными шаблонами обработки;
* иметь AI-помощника для анализа структуры Excel и подготовки инструкции;
* использовать локальную LLM опционально, а без модели работать через эвристики/fallback;
* хранить правила, историю и обучающие примеры;
* делать предпросмотр и экспорт результата.

Нужно не переписывать проект с нуля, а аккуратно улучшить текущую архитектуру, интерфейс, UX, документацию, обработку ошибок и демонстрационную готовность.

2. Главное позиционирование продукта

Переделай визуальную и текстовую подачу проекта так, чтобы он выглядел как продукт для компаний.

Рабочее позиционирование:

“Excel AI Normalizer — локальный ИИ-помощник для превращения сложных Excel-отчётов в нормализованные таблицы для 1С, Power BI, SQL, BI-аналитики и корпоративных хранилищ.”

Важно:

* не позиционировать продукт как “замену Excel”;
* не писать, что ИИ всегда гарантированно решает задачу;

8. Улучшить обработку ошибок

Ошибки должны быть понятны пользователю, а не выглядеть как traceback.

Для пользовательских ошибок показывать:

* что случилось;
* почему могло случиться;
* что сделать;
* пример корректной инструкции.

Примеры:

* файл не является Excel;
* файл слишком большой;
* не найден лист;
* не удалось определить заголовки;
* нет подходящих столбцов для melt;
* правило невалидно;
* LLM недоступна, используется fallback;
* файл повреждён;
* нет прав на запись результата.

В API ошибки должны возвращаться в JSON:

```json
{
  "error": "Краткое описание",
  "details": "Подробности",
  "suggestion": "Что сделать пользователю",
  "code": "ERROR_CODE"
}
```
