# Excel Converter / Excel Normalizer

Веб-приложение на Flask для преобразования неоднородных Excel-файлов в нормализованные плоские таблицы. Проект помогает разобрать структуру исходного файла, применить сохраненный шаблон обработки или сформировать новый шаблон по пользовательской инструкции.

## Возможности

- загрузка `.xlsx` и `.xls` через веб-интерфейс или API;
- конвертация без правила: чтение Excel и сохранение результата в `.xlsx`;
- работа с сохраненными шаблонами преобразования;
- AI-помощник для анализа структуры Excel и подготовки инструкции обработки;
- предпросмотр результата перед финальной конвертацией;
- диагностика ошибок преобразования с подсказками для уточнения инструкции;
- сохранение шаблонов, истории конвертаций и обучающих примеров;
- пакетная загрузка обучающих пар `source -> target`;
- защита от перезаписи загруженных файлов с одинаковыми именами;
- выгрузка результата в Excel с автофильтром, закреплением заголовков, итогами `SUBTOTAL` и сводным листом при наличии соответствующего правила.

## Быстрый Старт

Рекомендуемая версия Python: 3.11. На Python 3.12+ проект также может работать, но некоторые опциональные AI-зависимости могут устанавливаться сложнее.

Windows:

```powershell
py -3.11 -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
flask --app main run
```

Linux / macOS:

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
flask --app main run
```

После запуска откройте:

```text
http://127.0.0.1:5000
```

## Настройка

Основные параметры задаются через переменные окружения или файл `.env`:

```text
SECRET_KEY=change-me
FLASK_DEBUG=false
AI_BACKEND=llama_cpp
AI_MODEL_PATH=models/qwen2.5-3b-instruct-q4_k_m.gguf
AI_MAX_TRAINING_EXAMPLES=8
ENABLE_ASYNC=false
```

Локальная LLM не обязательна. Если модель не установлена, приложение использует встроенные эвристики и остается работоспособным.

Для включения локальной LLM установите дополнительную зависимость и положите GGUF-модель в `models/`:

```bash
pip install -r requirements-ai.txt
```

Быстрая установка рекомендуемой модели на Windows:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/download_model.ps1
```

Скрипт скачивает модель Qwen2.5 3B Instruct GGUF в путь:

```text
models/qwen2.5-3b-instruct-q4_k_m.gguf
```

После скачивания путь уже совпадает со значением `AI_MODEL_PATH` из `.env.example`.

`requirements-ai.txt` использует готовые CPU-колеса `llama-cpp-python==0.3.16`, чтобы на Windows не требовалась локальная сборка через Visual C++. Версия закреплена намеренно: на некоторых CPU более новая сборка может падать с `Windows Error 0xc000001d`.

## Модель И Обучение

В текущей поставке нет отдельного файла дообученной модели. Папка `models/` предназначена для внешней GGUF-модели, например Llama или Mistral, но сама модель не создается этим проектом и обычно передается отдельно из-за большого размера.

Проект использует два уровня логики:

- базовая логика и эвристики работают без LLM-модели;
- если в `models/` положить GGUF-файл и указать путь в `AI_MODEL_PATH`, AI-помощник сможет использовать локальную языковую модель;
- обучающие пары из `training_examples/` и правки из `instruction_feedback.json` используются как память примеров и few-shot-контекст, а не как fine-tuning весов.

Для полной передачи заказчику нужно включить:

- код проекта;
- `rules.json`;
- `training_examples.json`;
- папку `training_examples/`;
- `instruction_feedback.json`, если нужно сохранить накопленные пользовательские исправления;
- GGUF-модель отдельно, только если заказчику нужна работа локальной LLM без fallback-режима.

Если GGUF-модель передается отдельно, положите ее, например, в:

```text
models/qwen2.5-3b-instruct-q4_k_m.gguf
```

Или скачайте ее автоматически:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/download_model.ps1
```

и проверьте `.env`:

```text
AI_MODEL_PATH=models/qwen2.5-3b-instruct-q4_k_m.gguf
```

Если при запуске появляется ошибка вида `Windows Error 0xc000001d`, переустановите AI-зависимости:

```bash
pip install --force-reinstall -r requirements-ai.txt
```

В этой поставке проблема была исправлена переходом с `llama-cpp-python 0.3.23` на `0.3.16`.

## Основной Сценарий

1. Откройте раздел `AI-помощник`.
2. Загрузите Excel-файл.
3. Опишите задачу простыми словами, например: `заголовки на строке 4, данные начинаются со строки 5, итоги убрать, месяцы перенести в колонку Период`.
4. Проверьте, как система поняла структуру файла и будущий результат.
5. При необходимости уточните инструкцию и обновите предпросмотр.
6. Сохраните инструкцию как шаблон или выполните финальную конвертацию.
7. Скачайте готовый `.xlsx`.

Для повторяющихся форматов используйте раздел `Конвертация`: загрузите новый файл и выберите сохраненный шаблон.

## Типовые Правила

В проект уже добавлен демонстрационный набор из 15 типовых правил. Они нужны, чтобы заказчик мог сразу проверить конвертацию без ручного создания шаблонов:

- простая плоская таблица;
- таблица со служебными строками сверху;
- многоуровневые заголовки;
- месяцы или дни в колонках с преобразованием в длинный формат;
- продажи по филиалам с итогами;
- складские остатки;
- производственный отчет;
- табель сотрудников;
- банковская выписка;
- расходы по статьям;
- прайс-лист;
- реестр заказов;
- журнал качества;
- проектный план-факт;
- анкеты и опросы.

Дополнительно добавлен профильный нефтегазовый набор из 10 правил:

- суточная добыча по скважинам;
- фонд скважин;
- бурение и проходка;
- КРС и ПРС;
- простои скважин и оборудования;
- закачка воды по нагнетательным скважинам;
- лабораторное качество нефти;
- баланс транспорта нефти;
- месячная добыча в колонках с переводом в длинный формат;
- исследования скважин.

Если нужно восстановить демо-набор правил, выполните:

```bash
python scripts/seed_demo_rules.py
```

## Обучающие Примеры

Раздел `Обучение` хранит пары Excel-файлов:

```text
001_source.xlsx
001_target.xlsx
002_source.xlsx
002_target.xlsx
```

Также поддерживается ZIP-архив с папками:

```text
source/001.xlsx
target/001.xlsx
```

Обучающие примеры используются как контекст для похожих файлов. Это не переобучает веса модели, а помогает системе учитывать уже исправленные пользователем сценарии.

## API

Проверка доступности:

```http
GET /health
GET /api/ping
```

Загрузка файла:

```http
POST /api/uploads
Content-Type: multipart/form-data

file=<excel-file>
```

Ответ содержит `job_id`, который используется в дальнейших запросах.

Конвертация:

```http
POST /api/convert/<job_id>
Content-Type: application/json

{
  "schema": null,
  "options": {}
}
```

Конвертация по инструкции:

```http
POST /api/convert-with-instruction/<job_id>
Content-Type: application/json

{
  "instruction": "Заголовки на строке 4, данные со строки 5",
  "generated_rule": {}
}
```

Создание шаблона:

```http
POST /api/rules
Content-Type: application/json

{
  "name": "Типовой отчет",
  "prompt": "Инструкция обработки",
  "generated_rule": {}
}
```

Пакетная загрузка обучающих примеров:

```http
POST /api/training/batch
Content-Type: multipart/form-data

batch_zip=<zip-file>
```

Если файл по `job_id` не найден, API возвращает `404` и JSON с полем `error`.

## Структура Проекта

```text
app/
  routes/
    web.py                  # веб-страницы
    api.py                  # JSON/API маршруты
  services/
    conversion_service.py   # конвертация и сохранение результата
    table_structure_analyzer.py
    ai/
      instruction_assistant.py
      llm_client.py
      smart_converter.py
  utils/
    file_manager.py
    rules_store.py
    training_examples_store.py
    feedback_store.py
templates/                  # HTML-шаблоны Flask
input/                      # загруженные файлы и meta/*.json
output/                     # готовые Excel-файлы
training_examples/          # обучающие пары
models/                     # локальные GGUF-модели, не обязательны
```

## Данные И Файлы

- `rules.json` хранит шаблоны обработки.
- `history.json` хранит историю конвертаций.
- `training_examples.json` хранит метаданные обучающих примеров.
- `instruction_feedback.json` хранит обратную связь по уточнениям инструкций.
- `input/meta/*.json` связывает `job_id` с исходным именем файла.

Загруженные файлы сохраняются с префиксом `job_id`, поэтому одинаковые имена не перезаписывают друг друга.

## Примечания По Эксплуатации

- Для разработки включайте debug через `FLASK_DEBUG=true`.
- Для передачи заказчику оставляйте `FLASK_DEBUG=false`.
- `SECRET_KEY` в `.env` нужно заменить на собственное значение.
- Большие модели в `models/` обычно не включают в архив проекта и передают отдельно.
