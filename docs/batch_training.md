# Пакетное обучение модели

В проект добавлен массовый импорт обучающих примеров для ИИ-модуля. Теперь не нужно добавлять пары Excel-файлов по одной.

## Вариант 1. ZIP-архив

Откройте страницу **Обучение** и загрузите ZIP-архив. Система автоматически найдёт пары файлов и сохранит их в `training_examples/`.

Поддерживаемые структуры архива:

```text
training_examples/
  001_source.xlsx
  001_target.xlsx
  002_source.xlsx
  002_target.xlsx
```

или:

```text
source/
  001.xlsx
  002.xlsx
target/
  001.xlsx
  002.xlsx
```

Также поддерживаются русские суффиксы: `_исходная`, `_итоговая`, `_плоская`, `_результат`.

## Вариант 2. Несколько файлов без ZIP

На странице **Обучение** можно выбрать сразу несколько исходных и итоговых Excel-файлов. Пары сопоставляются по имени.

Примеры корректного сопоставления:

```text
01_report_source.xlsx  -> 01_report_target.xlsx
05_merged_source.xlsx  -> 05_merged_target.xlsx
```

## Защита от дублей

При добавлении считается SHA-256 исходного и итогового файла. Если такая же пара уже есть для того же правила, она будет пропущена.

## Переносимость проекта

Новые примеры сохраняются с относительными путями. Старые абсолютные Windows-пути автоматически восстанавливаются по ID файла внутри `training_examples/`.

## API

Для интеграции можно использовать эндпоинт:

```http
POST /api/training/batch
Content-Type: multipart/form-data

batch_zip=<zip-file>
rule_id=<optional>
prompt=<optional>
```
