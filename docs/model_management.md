# Управление локальными GGUF-моделями

## Назначение

SheetNorm сканирует каталог `AI_MODELS_DIR` и показывает найденные `.gguf` на странице `/settings`. Модель выбирается глобально для текущей установки: она загружается на сервере, а не в браузере пользователя.

## Добавление модели

1. Установите `llama-cpp-python`:

   ```powershell
   python -m pip install -r requirements-ai.txt
   ```

2. Поместите GGUF-файл в `models/` или во вложенный каталог.
3. Перезапустите SheetNorm или откройте `/settings` и нажмите **Обновить список**.
4. В автоматическом режиме SheetNorm сам назначит подходящую модель. Она загрузится в RAM только при первом AI-запросе.

Пример структуры:

```text
models/
  qwen/
    qwen2.5-coder-3b-instruct-q4_k_m.gguf
    qwen2.5-coder-3b-instruct-q4_k_m.json
  qwen2.5-coder-7b-instruct-q4_k_m.gguf
```


## Автоматический подбор

По умолчанию `AI_MODEL_SELECTION_MODE=auto`. При запуске SheetNorm:

1. сканирует `AI_MODELS_DIR`;
2. определяет CPU, доступную RAM, лимит контейнера и NVIDIA GPU;
3. исключает модели, которые не оставляют безопасный запас памяти;
4. выбирает наиболее мощную из подходящих;
5. сохраняет её как модель для AI-запросов;
6. загружает GGUF лениво при первом обращении к AI;
7. выполняет короткую проверку только если файл или параметры изменились.

Основные параметры:

```env
AI_MODEL_SELECTION_MODE=auto
AI_AUTO_SELECT_ON_STARTUP=true
AI_AUTO_ACTIVATE=true
AI_AUTO_TEST=true
AI_RESELECT_IF_UNAVAILABLE=true
AI_MIN_FREE_RAM_GB=2
AI_MAX_RAM_USAGE_RATIO=0.72
```

`AI_MIN_FREE_RAM_GB` задаёт обязательный запас для Windows, pandas и Excel-файлов. `AI_MAX_RAM_USAGE_RATIO` ограничивает долю доступной RAM, которую может занять модель. Если ни одна GGUF не подходит, приложение остаётся работоспособным через fallback.

Ручной режим остаётся доступен администратору на странице `/settings`.

## Sidecar metadata

Рядом с GGUF можно положить JSON с тем же именем:

```json
{
  "display_name": "Qwen 2.5 Coder 7B Q4_K_M",
  "family": "Qwen 2.5 Coder",
  "parameters_b": 7,
  "quantization": "Q4_K_M",
  "minimum_ram_gb": 8,
  "recommended_ram_gb": 12,
  "recommended_context": 4096,
  "description": "Сбалансированная модель для инструкций и pandas-кода"
}
```

Без sidecar семейство, число параметров и квантизация приблизительно извлекаются из имени файла.

## Профили

- **Экономичный**: 2048 токенов, небольшой batch, до 4 CPU-потоков.
- **Сбалансированный**: 4096 токенов, до 8 CPU-потоков.
- **Производительный**: 8192 токенов, увеличенный batch, GPU offload при обнаруженном NVIDIA GPU.
- **Пользовательский**: ручная настройка `n_ctx`, `n_threads`, `n_batch`, `n_gpu_layers` и temperature.

Рекомендация по памяти приблизительная. Фактическое потребление зависит от модели, окна контекста, сборки llama.cpp и GPU offload. Перед активацией всегда выполняется тест.


## Управление оперативной памятью

GGUF загружается лениво: открытие сайта, предпросмотр Excel и обычная конвертация по сохранённому правилу не должны загружать LLM. Модель появляется в RAM только при реальном AI-запросе.

Доступны режимы:

- **Экономичный** — после завершения AI-запроса модель автоматически выгружается через `AI_IDLE_UNLOAD_SECONDS`; рекомендуется для ПК с 8–16 ГБ RAM.
- **Быстрый** — модель остаётся в памяти для следующих запросов.
- **Вручную** — модель остаётся в памяти, пока администратор не нажмёт «Выгрузить модель из RAM».

Рекомендуемые локальные параметры:

```env
ASYNC_MODE=thread
LOCAL_WORKER_THREADS=1
AI_MEMORY_MODE=economy
AI_IDLE_UNLOAD_SECONDS=300
```

На странице настроек отображаются PID процесса, его RSS-память и активные фоновые задачи. Если в Windows видны две копии Python с загруженной моделью, проверьте, что Flask reloader отключён и не запущены несколько экземпляров приложения.

## Безопасное переключение

- В автоматическом режиме модель назначается без ручной кнопки и проверяется при первом AI-запросе.
- Назначенная модель может иметь состояние «не загружена в RAM» — это штатный lazy loading.
- Повторная проверка выполняется только после изменения файла модели или параметров запуска.
- Если автоматическая загрузка не удалась, SheetNorm пробует следующую подходящую модель.
- В ручном режиме новая модель активируется после теста, а прежняя сохраняется при ошибке.
- Режим fallback можно включить в любой момент.

## Где хранятся настройки

- При `DATA_STORE_BACKEND=database` — таблица `ai_settings`.
- В JSON-режиме — `AI_SETTINGS_FILE`, по умолчанию `ai_settings.json`.
- `.env` используется только для первоначальных значений и не редактируется из интерфейса.

После обновления существующей PostgreSQL-инсталляции выполните:

```bash
flask --app main db upgrade
```

## Docker

Каталог моделей монтируется read-only:

```yaml
- ./models:/app/models:ro
```

GGUF не копируются внутрь Docker image. Web и worker должны получать одинаковый `AI_MODELS_DIR` и доступ к общей папке моделей.

## Типичные ошибки

### `llama-cpp-python не установлен`

Установите `requirements-ai.txt` в том же окружении, где запущен SheetNorm.

### `Может не хватить памяти`

Выберите 3B вместо 7B, более сильную квантизацию или экономичный профиль.

### Файл виден, но тест не проходит

Проверьте целостность GGUF, совместимость версии llama.cpp, доступную RAM и параметры GPU offload.

### Модель удалена после активации

SheetNorm покажет ошибку загрузки и продолжит работать через fallback для новых AI-запросов. Выберите другую модель в настройках.
