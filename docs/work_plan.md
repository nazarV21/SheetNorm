## План работ по веб-конвертеру Excel → плоские таблицы

### 1. Цели и ограничения
- Обеспечить загрузку файлов Excel/CSV через веб-интерфейс и преобразование в стандартизованные таблицы.
- Поддержать большие объёмы данных: потоковая обработка, постраничная выдача результатов, фоновые задачи.
- Встроить модуль ИИ, способный предлагать сопоставление столбцов и готовые пресеты для преобразования. Требование: использовать открытые модели/решения, доступные в РФ (например, локальный `llama.cpp`, [Mistral](https://mistral.ai/), `rwkv`, `saiga`, `GigaChat` при наличии лицензии).
- Реализовать ролевую модель (минимум аутентификация сотрудников организации).

### 2. Архитектура на уровне компонентов
1. **Frontend (позже)**: браузерное SPA/MPA, взаимодействующее с REST API.
2. **Flask API**  
   - `routes.api` — загрузка файлов, управление шаблонами, получение результатов.  
   - `routes.health` — мониторинг.
3. **Сервис преобразования (`services.conversion`)**  
   - Чтение Excel (pandas/openpyxl).  
   - Применение пайплайна нормализации (валидация, типизация, переименование, агрегирование).  
   - Экспорт в CSV/Parquet/SQL.
4. **AI-помощник (`services.ai`)**  
   - Анализ примеров входных таблиц.  
   - Генерация mapping'а столбцов и правил очистки.  
   - Возможность выбора движка:  
     - локальный `llama_cpp` с GGUF-моделями (Llama 3, Mistral).  
     - внешние API (например, `GigaChat`, `Mistral`, `DeepSeek`) через адаптеры.
5. **Хранилище**  
   - Файловая система (`input/`, `output/`) для артефактов.  
   - Позже: Postgres для метаданных, Redis для очередей.
6. **Очереди/фон**  
   - Celery/RQ + Redis для тяжёлых задач.  
   - WebSocket/Server-Sent Events для прогресса.

### 3. Поток данных
1. Пользователь загружает Excel → `/api/uploads`.
2. Файл попадает в `input/`, создаётся запись задания.
3. ИИ анализирует первые строки, предлагает схему.
4. Пользователь подтверждает/корректирует схему.
5. Запускается конвертация → результат в `output/`.
6. Пользователь скачивает итог или отправляет в БД.

### 4. Модуль ИИ — стратегии
- **Подготовка данных**: выборка заголовков, статистика типов, примеры значений.
- **LLM-промпт**: контекст (бизнес-правила, формат целевой таблицы) + образцы входных строк.
- **Evaluations**: авто-сравнение предложенных правил с эталонными пресетами.
- **Кеширование**: сохранять подсказки ИИ для повторного использования одинаковых шаблонов.

### 5. Технические вехи
| Этап | Содержание | Выход |
|------|------------|-------|
|M1|Базовый Flask (аплоад, health, структура сервисов)|Рабочий сервер|
|M2|Пайплайн конвертации (валидация, нормализация, экспорт)|API `/convert`|
|M3|Интеграция ИИ (адаптер, промпты, UI для подтверждения)|AI рекомендации|
|M4|Фоновые задачи + мониторинг|Очереди, метрики|
|M5|Интеграция с корпоративными системами (SSO, storage)|Готовность к деплою|

### 6. Набор технологий
- Python 3.11+, Flask, Marshmallow/Pydantic, Pandas, PyArrow.
- Gunicorn + gevent/uvicorn-worker.
- `llama-cpp-python` или `transformers` + `optimum` для локальных моделей.
- Celery/RQ, Redis, PostgreSQL (позже).
- Docker + docker-compose для развёртывания.
